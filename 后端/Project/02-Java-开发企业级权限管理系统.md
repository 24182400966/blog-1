# Java 开发企业级权限管理系统

RBAC（Role-Based Access Control）支持三个著名的安全原则：最小权限原则、责任分离原则和数据抽象原则。

- 最小权限原则：RBAC 可以将其角色配置成其完成任务所需要的最小的权限集。
- 责任分离原则：通过调用相互独立互斥的角色来共同完成敏感的任务而体现。
- 数据抽象原则：可以通过权限的抽象来体现，通过操作抽象权限，而不用操作系统提供的典型的读、写、执行权限。
